## 1. 새 시스템이 돌아가는 방식 (요약)

### 1.1 턴 순서 결정

매 턴 사이클 시작 시, 각 캐릭터의 **SPD(속도)**로 행동 순서를 계산합니다.

```
행동 간격 = 300 / SPD
```

- SPD가 높을수록 행동 간격이 짧아져 **한 사이클에 더 많은 턴**을 가짐
- 예시: SPD 100이면 3턴, SPD 50이면 1턴, SPD 30이면 1턴

### 1.2 팀 공용 SP

- 아군/적군 각각 독립된 SP 풀 (최대 2000)
- **해당 팀** 행동 시에만 충전
- 궁극기를 쓰면 해당 팀의 SP에서 캐릭터별 비용만큼 차감
- 라운드가 끝나면 SP는 0으로 초기화

### 1.3 자동 궁극기

- 일반 공격과 액티브 스킬은 **항상 자동** (수동 선택 불가)
- "자동 모드"는 **궁극기만** 자동/수동 전환
- 적팀은 항상 자동
- 아군은 유저가 자동/수동 선택 가능

### 1.4 한 턴의 흐름

```
① 데빌 행동
② 차일드들 행동 순서 정렬
③ 순서대로 한 명씩 행동:
   → 행동 성공 시 해당 팀 SP 충전
   → 반격(카운터) 발생 여부 체크
   → 궁극기 발동 가능 여부 체크 (SP 충분하면 발동)
   → 궁극기/카운터가 새로 추가되면 순서를 다시 정렬
④ 모든 행동 완료 후 턴 종료
```

**핵심:** 궁극기와 카운터는 **"끼어들기"** 방식으로, 일반 행동 순서보다 **항상 먼저** 실행됩니다.

## 2. 검증한 상황들

| 상황 | 결과 |
| --- | --- |
| 일반 행동 → SP 충전 → 궁극기 자동 발동 | **정상** |
| 같은 캐릭터가 궁극기를 2번 연속 쓰는 경우 | **불가능** (쿨타임으로 차단) |
| 궁극기 끼어들기가 무한 반복되는 경우 | **불가능** (최대 100회 제한) |
| 속도 버프/디버프 후 턴 수 변화 | **정상** (매번 현재 속도로 재계산) |
| 수동 궁극기 대기 중 턴/라운드가 끝난 경우 | **정상** (SP 자동 환불) |
| 자동↔수동 전환 중간에 대기 중인 궁극기 | **정상** (전환 시 즉시 환불) |
| 카운터(반격)와 SP의 관계 | **독립** (카운터는 SP 안 씀) |
| 라운드가 바뀔 때 SP 상태 | **정상** (0으로 초기화) |

## 3. 빠른 캐릭터가 연속으로 행동하는 패턴

### 3.1 구체적 예시

**캐릭터 설정:**

- 아군 Fast: SPD=100, 턴당 SP 충전=60, 궁극기 비용=100
- 아군 Slow: SPD=50, 턴당 SP 충전=40, 궁극기 비용=80
- 적군 Enemy: SPD=75, 턴당 SP 충전=50, 궁극기 비용=120

**한 사이클 내 행동 횟수:**

- Fast: 3턴 (SPD가 높아서)
- Enemy: 2턴
- Slow: 1턴

### 3.2 실제 행동 순서 추적

| 순서 | 누가 | 무엇을 | SP 변화 (아군) |
| --- | --- | --- | --- |
| 1 | Fast | 일반 공격 | 0 → 60 |
| 2 | Enemy | 일반 공격 | 60 → 110 |
| - | - | *SP 110 ≥ Fast 궁극기 비용 100 → 궁극기 발동!* | 110 → 10 |
| 3 | **Fast** | **궁극기** (끼어들기) | 10 → 70 |
| - | - | *적팀도 SP 충분 → Enemy 궁극기 발동* | - |
| 4 | Enemy | 궁극기 (끼어들기) | 70 → 120 |
| - | - | *SP 120 ≥ Slow 궁극기 비용 80 → 발동!* | 120 → 40 |
| 5 | Slow | 궁극기 (끼어들기) | 40 → 80 |
| 6 | Fast | 일반 공격 | 80 → 140 |
| 7 | Slow | 일반 공격 | 140 → 180 |
| 8 | Enemy | 일반 공격 | 180 → 230 |
| 9 | Fast | 일반 공격 | 230 → 290 |

**결과:** Fast는 4번, Enemy는 3번, Slow는 2번 행동

### 3.3 주목할 점

- Fast가 일반 공격 직후 바로 궁극기를 사용 → **연속 2회 행동**
- 이는 궁극기가 "끼어들기"로 일반 순서보다 먼저 실행되기 때문
- **무한 반복은 아님** — 같은 캐릭터의 궁극기는 한 사이클에 1회만 가능 (쿨타임)

## 4. 기존 시스템과의 주요 차이점

|  | 기존 (라운드로빈) | 새 시스템 (타임라인) |
| --- | --- | --- |
| **턴 수** | 모든 캐릭터 동일하게 1턴씩 | SPD에 비례 (빠를수록 더 많은 턴) |
| **SPD의 역할** | 행동 순서만 결정 | 순서 + 턴 수 + SP 기여도 모두 결정 |
| **SP 축적 패턴** | 캐릭터 수 × 1턴씩 균등 | 빠른 캐릭터가 더 많이 기여 |
| **궁극기 타이밍** | 비교적 균등하게 모임 | 빠른 팀이 먼저 SP를 모아 먼저 발동 |
| **속도 버프 효과** | 순서가 앞당겨짐 | 순서 앞당겨짐 + **턴 수 자체가 증가** |
| **SPD 스탯 가치** | 보통 | **매우 높음** |

## 5. 밸런스 관점에서 주의할 점

### 주의 1: 궁극기 자동 발동 시 우선순위 처리 방식

현재 자동 궁극기는 **우선순위가 높은 캐릭터부터 SP를 소모**합니다.
만약 1순위 캐릭터의 궁극기 비용이 현재 SP보다 크면, **2순위 이하 캐릭터들도 전부 발동하지 않습니다.**

**예시:**

```
현재 SP: 150
1순위 캐릭터 A: 궁극기 비용 200 → SP 부족으로 발동 실패
2순위 캐릭터 B: 궁극기 비용 80  → A가 실패했으므로 체크조차 안 됨!

→ SP가 150이나 있는데 아무 궁극기도 안 나감
```

이것은 "1순위 캐릭터를 위해 SP를 아껴두는" 설계입니다.
만약 "쓸 수 있는 캐릭터는 다 쓰게" 하려면 로직 변경이 필요합니다.

**두 가지 방식 비교:**

|  | 현재 방식 (대기) | 변경 시 (건너뛰기) |
| --- | --- | --- |
| **동작** | 1순위 실패 시 전원 대기 | 1순위 실패 시 2순위로 넘어감 |
| **SP 활용도** | 낮음 (남는 SP 발생) | 높음 (쓸 수 있으면 다 씀) |
| **궁극기 총 횟수** | 적음 | 많음 |
| **유저 우선순위** | 엄격히 존중 | 느슨하게 존중 |
| **고비용 캐릭터** | 유리 (SP 독점 대기) | 불리 (저비용이 먼저 가져감) |
| **전투 템포** | 느림 | 빠름 |
| **적팀도 동일 적용** | 적 궁극기도 적게 나옴 | 적 궁극기도 많아짐 |

### 주의 2: SPD 스탯의 가치 급상승

새 시스템에서 SPD는 단순히 "누가 먼저 행동하나"가 아니라:

1. **행동 횟수 자체를 결정** (SPD 100 → 3턴, SPD 50 → 1턴)
2. **더 많은 행동 = 더 많은 SP 충전** → 궁극기를 먼저 쓸 수 있음
3. **더 많은 행동 = 더 많은 데미지/힐** → 직접적 전투력 증가

→ SPD 밸런스가 매우 중요해졌습니다. SPD 차이가 곧 행동 수 차이이므로,
SPD가 지나치게 높은 캐릭터가 게임을 지배할 수 있습니다.

### 주의 3: 속도 버프/디버프의 영향이 극적

기존에는 속도 버프가 "순서를 앞당기는" 정도였지만, 새 시스템에서는:

- **속도 2배 버프 → 남은 턴 수가 2배로 증가** → SP 충전도 2배 → 궁극기도 더 빨리
- **속도 절반 디버프 → 남은 턴 수가 절반으로 감소**

속도 관련 스킬(버프/디버프)의 영향력이 이전보다 훨씬 크므로, 밸런스 점검이 필요합니다.
