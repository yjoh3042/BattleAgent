## 3. 속성 상성 비교 — ElementUpper.xlsx 기준

### 3게임 속성 상성 비교

| 게임 | 기본 3원소 순환 | 빛/어둠 관계 | 피해 배율 |
|------|:---:|:---:|:---:|
| **서머너즈워** | 불>풍>물>불 | 빛↔어둠 상호 유리 | 유리 +50% / 불리 -50% |
| **에픽세븐** | 불>얼음>대지>불 | 빛↔어둠 상호 불리 | 유리 +20% / 적중 ±10% |
| **스타레일** | 7속성 약점격파 | 양자/허수 별도 | 약점 속성만 내구도 감소 |
| **BattleAgent** | **Water>Fire>Forest>Water** | **Light↔Dark 상호 +20%** | **유리 +20% / 불리 -20%** |

### ElementUpper.xlsx 원본 데이터

| ID | Element | Win (유리 속성) | Lose (불리 속성) |
|:---:|:---:|:---:|:---:|
| 1 | **Water** | Fire | Forest |
| 2 | **Fire** | Forest | Water |
| 3 | **Forest** | Water | Fire |
| 4 | **Light** | Dark | Dark |
| 5 | **Dark** | Light | Light |

### 구현된 상성 배율 매트릭스 (damage_calc.py ELEMENT_TABLE)

```
공격↓ 방어→   Water   Fire   Forest  Light   Dark
  Water       1.0    1.2↑    0.8↓    1.0     1.0
  Fire        0.8↓   1.0     1.2↑    1.0     1.0
  Forest      1.2↑   0.8↓    1.0     1.0     1.0
  Light       1.0    1.0     1.0     1.0     1.2↑
  Dark        1.0    1.0     1.0     1.2↑    1.0
```

| 상성 유형 | 배율 | 해당 조합 |
|:---:|:---:|------|
| **Win (유리)** | **×1.2 (+20%)** | Water→Fire, Fire→Forest, Forest→Water, Light→Dark, Dark→Light |
| **Lose (불리)** | **×0.8 (-20%)** | Fire→Water, Forest→Fire, Water→Forest |
| **Light↔Dark** | **양쪽 ×1.2** | 서로에게 Win이면서 Lose → 하이리스크·하이리턴 |
| **중립** | **×1.0** | 상성 관계 없는 조합 |

### 상성 다이어그램

```
        Water 💧
       ╱  1.2  ╲
  0.8 ↓         ↑ 0.8
     ╱    1.2    ╲
 Fire 🔥 ──────▶ Forest 🌿
         ◀──────
          0.8

   Light ✨ ◀──1.2──▶ Dark 🌑
            ◀──1.2──▶
   (양방향 유리 = 서로 폭딜)
```

### BattleAgent Light↔Dark 차별점

| 게임 | 빛↔어둠 관계 | 전략적 의미 |
|------|:---:|------|
| 서머너즈워 | 상호 유리 (+50%) | 양쪽 모두 강한 공격 (배율 큼) |
| 에픽세븐 | 상호 불리 (-20%) | 양쪽 모두 약한 공격 |
| 스타레일 | 별도 속성 체계 | 약점격파 기반 |
| **BattleAgent** | **상호 유리 (+20%)** | **양쪽 모두 20% 보너스 = 서로 폭딜하는 하이리스크 구조** |

## 4. BattleAgent 최대 차별점: 공용 SP 얼티밋 시스템

### 3게임의 궁극기 자원 비교

| 항목 | 서머너즈워 | 에픽세븐 | 스타레일 | **BattleAgent** |
|------|:---:|:---:|:---:|:---:|
| **궁극기 자원** | 쿨턴 (개인) | 쿨턴 (개인) | 에너지 (개인) | **공용 SP (팀 공유)** |
| **자원 범위** | 캐릭터별 독립 | 캐릭터별 독립 | 캐릭터별 독립 | **진영 전체 공유 풀** |
| **최대치** | 스킬별 쿨턴 | 스킬별 쿨턴 | 개인 에너지 Max | **MAX_SP = 10** |
| **충전 방식** | 턴 경과 시 쿨턴 -1 | 턴 경과 시 쿨턴 -1 | 공격/피격/스킬로 충전 | **매 턴 양 진영 +1** |
| **소비 경쟁** | 없음 (개인) | 없음 (개인) | 없음 (개인) | **팀 내 경쟁** |
| **전략 깊이** | 스킬 로테이션 | 스킬 로테이션 | 에너지 관리 | **자원 배분 의사결정** |

### 공용 SP 시스템 상세 (sp_manager.py)

```
┌─────────────────────────────────────────────────────────┐
│              공용 SP 풀 (MAX = 10)                       │
│                                                         │
│   매 턴 +1 충전 (피아 구분 없이 누군가 행동할 때마다)      │
│                                                         │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐                │
│   │ATTACKER │  │MAGICIAN │  │SUPPORTER│                │
│   │ ULT: 6  │  │ ULT: 4  │  │ ULT: 4  │                │
│   └────┬────┘  └────┬────┘  └────┬────┘                │
│        │            │            │                      │
│        └────────────┼────────────┘                      │
│                     ▼                                   │
│          ❓ 누구의 얼티밋을 쓸 것인가?                    │
│                                                         │
│   SP=6 → ATTACKER 궁 (폭딜)  vs                        │
│   SP=4 → MAGICIAN 궁 (CC) + 남은 SP 2 저축             │
│   SP=3 → HEALER 궁 (회복) + SP 7 남겨서 다음턴 ATK 궁   │
│                                                         │
│         ★ "누구에게 쓰느냐"가 승패를 결정 ★              │
└─────────────────────────────────────────────────────────┘
```

### 역할별 SP 비용과 전략

| 역할 | SP 비용 | 필요 턴 수 | 전략적 의미 |
|:---:|:---:|:---:|------|
| **ATTACKER** | **6** | 6턴 | 가장 비쌈 → 타이밍이 핵심, "언제 폭딜할 것인가" |
| **MAGICIAN** | **4** | 4턴 | 중간 → CC 선제 투입 유리, ATK보다 2턴 빠름 |
| **SUPPORTER** | **4** | 4턴 | 중간 → 버프 궁과 CC 궁이 SP 경쟁 |
| **HEALER** | **3** | 3턴 | 저렴 → 긴급 회복에 빠르게 투입 가능 |
| **DEFENDER** | **3** | 3턴 | 저렴 → 방어 궁 선사용 후 SP 축적 전략 |

### 3게임에 없는 BattleAgent만의 전략적 딜레마

| 딜레마 | 설명 |
|------|------|
| **궁 타이밍 경쟁** | ATTACKER 궁(6SP) 1회 vs HEALER 궁(3SP)+MAGICIAN 궁(4SP)=7SP 2회. 한 번의 폭딜 vs 두 번의 유틸리티 |
| **선투자 vs 저축** | SP=4 시점에서 MAGICIAN CC궁을 즉시 사용할지, 2턴 더 기다려 ATTACKER 폭딜궁을 사용할지 |
| **진영 간 SP 레이스** | 양 진영이 동시에 +1씩 충전 → 상대보다 먼저 궁을 써서 이니셔티브 장악 |
| **SP 갈취 전략** | SP 증감 스킬로 상대 SP를 깎거나 아군 SP를 추가 충전 → 궁 타이밍 역전 |
| **조합 SP 예산** | 5인 파티에서 ATK 2명(6+6=12SP) 넣으면 둘 다 궁 못 씀 → 역할 배분이 SP 경제와 직결 |

          ┌──────────────────────────────────────────┐

          │                        │                 │
          │                    카운터                 │
          │                        ▼                 │
          │                  ② CC 잠금               │
          │                        │                 │
          │                    카운터                 │
          │                        ▼                 │
          │                  ④ 클렌즈/면역            │
          │                        │                 │
          │                    카운터                 │
          │                        ▼                 │

| **M02 독정원** | 64.8% | 서머너즈워 면역+DoT | ✅ 독 DoT도 안정적 상위 |
| **M03 출혈폭딜** | 60.1% | 스타레일 격파팀(화류) | ✅ DoT+버스트 하이브리드 |
| **M04 CC잠금** | 56.1% | 에픽세븐 CC체인 | ✅ 적절한 중위권 (카운터 존재) |

| **M10 야성해방** | 50.4% | 스타레일 하이퍼캐리 | ✅ 하이퍼캐리는 편차 큰 50%대 |
| **M05 속도처형** | 4.7% | 3게임 속도 선공 | ❌ **비정상** — 속도 메타가 최하위 |
| **M06 크리폭딜** | 7.2% | 3게임 버스트딜 | ❌ **비정상** — 크리 버스트 너무 약함 |
| **M07 방어무력** | 15.4% | 스타레일 은늑대 방감 | ⚠️ 방감 전략 약한 건 정상이나 너무 낮음 |

### SP 경제가 카운터 순환에 미치는 영향

```
DoT 메타(M01/M02)가 강한 이유:
→ 노멀/액티브 스킬로 DoT를 걸고, SP는 CC궁에 투자
→ SP를 "안 써도 이기는" 구조 = SP 효율 최강

하이퍼캐리(M10)가 약한 이유:
→ ATK 3명의 궁(6×3=18SP)을 전부 쓸 수 없음
→ SP 병목으로 화력이 기대치의 60%만 발휘
```

### BattleAgent만의 차별화 요소 (Top 5)

| 순위 | 차별점 | 설명 |
|:---:|------|------|
| **1** | **공용 SP 얼티밋** | 3게임 모두 궁극기가 "개인 자원"이지만, BattleAgent만 **팀 공유 자원** 사용. "누구의 궁을 쓸 것인가"라는 팀 단위 의사결정 요구 |
| **2** | **역할별 SP 비용 차등** | ATK=6, MAG/SUP=4, HEA/DEF=3으로 역할이 SP 경제를 결정 → "화력 vs 유틸리티" 트레이드오프 자동 발생 |
| **3** | **Light↔Dark 상호 폭딜** | ElementUpper에서 빛/어둠이 서로 Win+Lose → 다른 게임의 "한쪽 유리" 구조와 다른 양방향 하이리스크 |
| **4** | **매 턴 양 진영 동시 충전** | 양 진영이 동일 속도로 SP 축적 → "누가 먼저 궁을 쓰느냐"의 레이스 |
| **5** | **SP 풀 라운드 리셋** | 배틀 라운드 전환 시 SP 0으로 초기화 → 라운드 내 즉시 판단 강제 |
