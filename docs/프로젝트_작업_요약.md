## 1. 프로젝트 개요

BattleAgent는 모바일 게임의 전투 시뮬레이션 엔진으로, CTB(Charge Time Battle) 턴제 기반의 5vs5 파티 대전을 구현한다.

- **엔진:** Python 기반, `BattleEngine` 코어
- **캐릭터 수:** 70명 등록 (34명 스킬 구현, 36명 스탯만)
- **테스트:** 53개 전체 통과
- **데이터 소스:** Excel(Character.xlsx, Skill.xlsx), Python(test_data.py), Notion DB

## 2. 핵심 시스템 구조

### 2.1 CTB 턴 시스템

```
행동 간격 = BATTLE_LENGTH(300) / SPD
```
- `heapq` 기반 우선순위 큐로 턴 순서 결정
- SP 공유 (파티 전원 합산), 얼티밋 끼어들기 지원

### 2.2 역할별 기본 스탯 (불변 룰)

| 역할 | SPD | SP 비용 |
|------|-----|---------|
| Attacker | 80 | 6 |
| Healer | 90 | 3 |
| Magician | 100 | 4 |
| Defender | 110 | 3 |
| Supporter | 120 | 4 |

### 2.3 전체 캐릭터 공통 스탯 (불변 룰)

> **2026-03-13 확정** — 아래 스탯은 모든 캐릭터에 동일 적용되며 개별 예외를 허용하지 않는다.

| 스탯 | 고정값 | 비고 |
|------|--------|------|
| `cri_ratio` (크리티컬 확률) | **0.15 (15%)** | StatBlock 기본값, 개별 설정 금지 |
| `penetration` (관통) | **0.0 (0%)** | StatBlock 기본값, 개별 설정 금지 |
| `cri_dmg_ratio` (크리 배율) | 1.5 (150%) | StatBlock 기본값 |
| `acc` (명중률) | 1.0 (100%) | StatBlock 기본값 |
| `dodge` (회피율) | 0.0 (0%) | StatBlock 기본값 |
| `cri_resist` (크리 저항) | 0.0 (0%) | StatBlock 기본값 |

**참고:** 스킬 버프/디버프를 통한 전투 중 cri_ratio/penetration 변동은 허용. 금지 대상은 **초기 스탯(StatBlock 생성자)** 에 한정.

### 2.4 전투 필드

- 3x3 그리드 배치 (row 0=전열, row 2=후열)
- `ENEMY_NEAR` 거리 = `caster.row + target.row + |caster.col - target.col|`
- 도발(Taunt) → 단일/다중 타겟 강제

### 2.5 속성 상성

| 속성 | 유리 | 불리 |
|------|------|------|
| Fire(화) | Forest(목) | Water(수) |
| Water(수) | Fire(화) | Forest(목) |
| Forest(목) | Water(수) | Fire(화) |
| Light(광) | Dark(암) | Dark(암) |
| Dark(암) | Light(광) | Light(광) |

### 2.6 Prefab 네이밍 규칙

- vidx: `c000` ~ `c999` (캐릭터 고유 번호)
- skeleton idx: `a` ~ `z`
- skin idx: `00` ~ `99`

## 3. 완료 작업 #1: c600/c601 OP 재설계

### 3.1 c600 "루미나" (광/서포터)

**설계 의도:** 시트리 + 엘리시온 + 브라우니의 장점을 통합한 최상위 서포터

| 항목 | 기존 | OP |
|------|------|-----|
| 이름 | c600 | 루미나 |
| ATK | 190 | 230 |
| DEF | 138 | 160 |
| HP | 5600 | 6800 |
| SPD | 120 | **120** (서포터 룰 준수) |
| Normal | 1.00x 단일 | 1.20x 단일 |
| Active | 최저HP 힐20%+DEF+20% | **전체 ATK+30%(3T) + SPD+20(3T) + 디버프 제거** |
| Ultimate | 전체 DEF+20%+힐15% (SP4) | **전체 힐35% + ATK+35%(3T) + CRI_DMG+0.50(3T) + SP+1** (SP**3**) |

**OP 핵심:**
- SPD 120 (서포터 룰) → 서포터 중 동률 최선행, 딜러 턴 전에 버프 보장
- Active 디버프 제거 내장 → 힐러 없이도 정화 가능
- Ult SP3으로 저렴하면서 시트리(ATK+CRI_DMG) + 엘리시온(힐+SP) 통합

### 3.2 c601 "에레보스" (목/공격자)

**설계 의도:** 이브 + 쿠바바 + 아르테미스의 장점을 합친 최상위 딜러

| 항목 | 기존 | OP |
|------|------|-----|
| 이름 | c601 | 에레보스 |
| 속성 | - | **Forest(목)** |
| ATK | 355 | **530** (게임 내 최고) |
| DEF | 115 | 135 |
| HP | 4900 | 5500 |
| SPD | 80 | **85** (공격자 중 최고) |
| CRI | 5% | ~~35%~~ → **15%** (룰 통일) |
| PEN | 0 | ~~0.20~~ → **0%** (룰 통일) |
| Normal | 2.00x 단일 | 2.50x x **2 hit** (실질 5.0x) |
| Active | 랜덤2 2.20x | **최저HP 5.00x + HP<50%시 +3.00x + DEF-25%** |
| Ultimate | 전체 1.30x (SP6) | **전체 3.50x + DEF-30%(2T) + 출혈DoT(0.20,3T)** (SP**5**) |

**OP 핵심:**
- ATK 530 (게임 내 최고) + SPD 85 (공격자 중 최고) = 기본 스탯 압도
- Normal 2연타 = 크리 기회 2배
- Active 처형 조건: HP<50% (이브의 30%보다 관대) → 총 8.0x + DEF-25% 디버프
- Ult AoE 3.50x + DEF-30%(2T) + 출혈DoT(20%,3T) → 후속 딜러 시너지
- ~~CRI 35% + PEN 20%~~ → 밸런스 룰 통일로 15%/0%로 조정 (2026-03-13)

### 9.2 기타 오류

|------|------|------|

| Python 한글 출력 깨짐 | Windows 콘솔 인코딩 | `PYTHONIOENCODING=utf-8` 환경변수 설정 |
