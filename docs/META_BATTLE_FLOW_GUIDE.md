## 📐 전투 시스템 개요

### CTB (Conditional Turn-Based) 엔진 플로우

```
┌─────────────────────────────────────────────────────────┐
│                    전투 시작                               │
│  1. 모든 유닛 heapq 큐 등록 (action_time = 300 / SPD)     │
│  2. 전투 시작 트리거 발동                                  │
└────────────────────┬────────────────────────────────────┘
                     ▼
┌─────────────────────────────────────────────────────────┐
│               메인 루프 (최대 500턴)                       │
│                                                          │
│  while turn_count < 500:                                 │
│    ① heapq에서 가장 빠른 유닛 pop                         │
│    ② 타임오버 체크 (current_time > 300.0)                 │
│       → 공격덱: 패배 / 방어덱: 승리                        │
│    ③ 배틀 라운드 전환 (10 time unit마다)                   │
│       → SP 초기화, 얼티밋 플래그 리셋                      │
│    ④ Hard CC 체크 (Freeze/Stun/Sleep/Stone)               │
│       → 행동 불가, DoT 틱만 발동                           │
│    ⑤ Soft CC 체크 (Panic → 30% 확률 행동 실패)            │
│    ⑥ 턴 시작 틱 (DoT: 화상/독/출혈 데미지)               │
│    ⑦ 얼티밋 끼어들기 체크 (SP 충분 + 쿨다운 완료)         │
│       → Extra Turn으로 얼티밋 실행                        │
│    ⑧ 스킬 결정 (액티브 쿨다운 완료 → 액티브, 아니면 노멀) │
│    ⑨ 스킬 실행 + 킬 트리거                               │
│    ⑩ 턴 종료 틱 (버프 지속시간 감소)                      │
│    ⑪ 승리/패배 판정                                       │
│       → 적 전멸 = ALLY_WIN / 아군 전멸 = ENEMY_WIN        │
└─────────────────────────────────────────────────────────┘
```

### 핵심 수치 레퍼런스

| 항목 | 값 | 설명 |
|------|-----|------|
| 행동 간격 | `300 / SPD` | SPD 80 → 3.75, SPD 120 → 2.50 |
| 배틀 라운드 | 10 time unit | SP 초기화 + 얼티밋 플래그 리셋 |
| SP 충전 | 턴마다 +1 | 아군/적 별도 풀, 최대 10 |
| 크리티컬 | 15% 확률 × 1.5배 | CRI율/CRI피해 버프로 증가 가능 |
| DoT (화상/독/출혈) | maxHP × ratio | 턴 시작 시 틱, 최대 3스택 |
| CC 지속 | Hard: 행동불가 / Soft(Panic): 30% 실패 | |
| 타임오버 | 300.0 time | 공격덱=패배, 방어덱=승리 |

### SP 시스템 & 얼티밋 플로우

```
┌─────────────────────────────────────────────────────────┐
│                  SP (Special Point) 시스템               │
├─────────────────────────────────────────────────────────┤
│  ■ 공유 풀: 아군 5명이 하나의 SP 풀 공유 (적도 별도 풀)  │
│  ■ 충전: 피아 구분 없이 누군가의 턴 시작마다 양측 +1     │
│  ■ 최대: 10 SP (초과 충전 불가)                          │
│  ■ 초기화: 배틀 라운드 전환(10 time unit)마다 0으로 리셋  │
└─────────────────────────────────────────────────────────┘
```

**얼티밋 발동 플로우:**
```
  ① 일반 턴 시작 (SP +1 충전)
  ② 얼티밋 끼어들기 체크:
     └→ SP ≥ sp_cost? (역할별: ATK=6, MAG=4, DEF=3, HLR=3, SUP=4)
     └→ 이번 라운드 미사용?
     └→ 쿨다운 완료?
  ③ 조건 충족 시:
     └→ SP 소모 (공유 풀에서 차감)
     └→ Extra Turn 등록 (현재 턴 직후 끼어들기)
  ④ Extra Turn 실행:
     └→ 얼티밋 스킬 발동
     └→ SP 충전 없음 (엑스트라 턴은 +1 안 함)
     └→ 버프 틱/쿨다운 감소 없음
  ⑤ 일반 턴으로 복귀 → 노멀/액티브 실행
```

**SP 경제의 전략적 의미:**

| SP 비용 | 역할 | 라운드 내 발동 가능 턴 | 전략 |
|:-------:|------|:--------------------:|------|
| 3 SP | DEF, HLR | ~3턴차 | 빠른 발동 → 초반 방어/힐 확보 |
| 4 SP | MAG, SUP | ~4턴차 | 중반 발동 → CC/버프 타이밍 |
| 6 SP | ATK | ~6턴차 | 후반 발동 → 축적된 버프로 최대 딜 |

- **공유 풀의 딜레마**: ATK(6SP) 얼티밋 사용 시 같은 라운드 내 SUP(4SP) 얼티밋 발동 불가 → 순서 관리 필요
- **라운드 리셋 리스크**: SP 9에서 라운드 전환 → 0으로 초기화, 사용하지 못한 SP 낭비
- **SPD와 SP 시너지**: 고SPD 캐릭터가 먼저 행동 → SP 먼저 소모 → 저SPD 딜러의 얼티밋이 SP 부족으로 지연

### 데미지 공식

```
base = ATK × (1 - DEF/(ATK+DEF))
damage = floor(base × skillMult × elementMult × critMult)

속성 상성: 유리 1.2× / 불리 0.8× / 중립 1.0×
  화→목, 목→수, 수→화 (삼각)
  광↔암 (상호 1.2×)
```

## 🧊 M02 — 빙결감옥 (Freeze Prison)

### 참조 게임: 서머너즈워 — CC Lock

> **서머너즈워의 CC Lock팀**은 가니메데(쿨초기화)+하토르(수면)+오케아노스(기절)로
> 적을 완전히 행동불능 상태로 만든 뒤 딜러가 처리하는 전략이다.
> M02는 Freeze+Stun 이중 CC 사이클로 적 행동을 봉쇄하고,
> 에르제베트(6.50× 노멀)가 봉쇄 중 딜을 담당한다.

> 오네이로이는 SUP(서포터) 역할이지만 **ATK 400 + 2.50× Active + Sleep CC**로
> 실질적 딜러+CC 역할을 수행한다. SPD 120으로 팀에서 가장 먼저 행동하여
> Sleep으로 적 핵심 캐릭터를 선제 봉쇄한다.

## ⚖️ 메타 밸런스 5대 법칙

### 법칙 1: 힐러 필수
> 힐러가 없으면 0%가 된다. 모든 메타에 최소 1명의 실제 힐 스킬 보유자 필수.

### 법칙 2: 처형 > DoT
> HP% 기반 처형이 DoT 누적보다 효율적이다. (M03 55.2% > M01 48.5%)

### 법칙 3: 방어덱 생존
> 킬파워 없어도 방어덱에서는 생존=승리. (M05: 공격덱 Avg 49.6% — 방어 특화)

### 법칙 4: CC는 킬파워 동반 필수
> CC만으로는 이길 수 없다. DPS 동반 필요. (M02: CC+에르제베트 6.50×)

### 법칙 5: 완벽한 메타는 없다
> M09(71.1%)도 M02에 0%. 상성 기반 카운터픽이 핵심.
