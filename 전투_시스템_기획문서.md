## 1. 기획 의도

- 새롭고 신선한 VIEW와 아트 스타일로 소진된 방치형 전투와 다른 **Look & Feel** 제공
- 차일드 한 명 한 명의 캐릭터성을 보여주면서 화려하고 멋진 아트워크로 **양산형 저퀄리티 게임 느낌 탈피**
- 쉽고 편한 플레이 안에 **파고들 요소** (조합·상성·시너지·전략) 제공으로 전투 깊이 확보
- 보기 편하고 **보는 맛(전투 쾌감)**이 있는 동시에 다양한 전투 양상 제공
- 컷씬 등 **원작 리소스 & IP의 느낌**을 잘 살린 전투 연출
- 유저의 **수동 개입 지점** 제공 — "내가 잘했어"라는 감각을 만들어줄 수 있는 전투 시스템
- 명확한 전투 로직과 **리플레이 기능**으로 "왜 패배했는가"의 납득과 전략 최적화 과정 제공

## 3. 전투 구조 및 연출

### 3-1. 전투 구조 유형

#### ① 단일 전투
```
전투시작연출 → 전투 → 승리/패배 연출
```

#### ② 연속 전투 (2~4전 구성, 데이터로 조정 가능)
```
전투1 → (달리기 연출) → 전투2 → 보스등장연출 → 전투3 → 전투4 → 승리/패배 연출
```
> 연전 시 HP/SP 등 전투 정보 유지. 모든 전투 승리가 최종 승리 조건.
> 마지막 전투 기준 원하는 **Battle Spot**에서 전투 가능.

#### ③ 특수 전투 (레이드 등)
```
전투시작연출 → 기록형 전투 → 승리(기록) 연출
```

### 3-2. 전투 연출

| 연출 | 내용 |
|------|------|
| ① 보스 등장 연출 | 주요 보스전 전투 시작 전 삽입. 중복 시 스킵 가능. |
| ② Ultimate 컷인 연출 | 얼티밋 스킬 사용 시 컷인 + 원작 일러스트 + 속도감 연출. 중요도 낮은 정보는 사이드 슬라이드 컷인 활용. |
| ③ 승리/패배 연출 | 승리 시 캐릭터 정면 + 고유 모션 출력. Stage Clear UI 결합 (스테이지 클리어 + 보상 + 다음 전투 도전). |

## 7. CTB 턴 시스템

### 7-1. 시스템 개요

본 전투는 **CTB(Conditional Turn-Based)** 방식으로, SPD(속도)를 기반으로 행동 순서와 행동 횟수를 동시에 결정한다.
기존 라운드로빈(모든 캐릭터 1턴씩)과 달리 **속도가 높을수록 한 사이클에 더 많은 턴**을 가진다.

### 7-2. 턴 순서 계산

```
행동 간격 = 300 / SPD
```

| SPD | 행동 간격 | 한 사이클 내 행동 횟수 (예시) |
|-----|-----------|-------------------------------|
| 100 | 3.0 | 3턴 |
| 75 | 4.0 | 2턴 |
| 50 | 6.0 | 1턴 |
| 30 | 10.0 | 1턴 |

> SPD가 높을수록 짧은 간격으로 행동 → 한 사이클 내 더 많은 행동 획득

### 7-3. 배틀 라운드 (Battle Round)

- 전투 시간이 **10 단위(10, 20, 30...)** 에 도달할 때마다 새로운 배틀 라운드 시작
- 생존한 모든 캐릭터의 행동 게이지를 **현재 속도 기준으로 강제 재계산**

### 7-4. 시점(Timing) 정의

| 시점 | 정의 |
|------|------|
| **전투 시작** | 첫 번째 캐릭터가 턴을 획득하기 직전 |
| **턴 시작** | 캐릭터가 턴을 획득한 시점 → SP 획득 발생 |
| **턴 종료** | 캐릭터가 스킬을 사용한 후 (턴 소모 간주) |
| **전투 종료** | 아군 또는 적군 전멸 직후 |

### 7-5. 한 턴의 흐름

```
① 데빌 행동
② 차일드들 행동 순서 정렬 (SPD 순)
③ 순서대로 한 명씩 행동:
   → 행동 성공 시 해당 팀 SP 충전
   → 반격(카운터) 발생 여부 체크
   → 얼티밋 발동 가능 여부 체크 (SP 충분 시 발동)
   → 얼티밋/카운터가 새로 추가되면 순서를 다시 정렬
④ 모든 행동 완료 후 턴 종료
```

### 7-6. 추가 턴 (Extra Turn) — "끼어들기"

**획득 조건**: "반격(Counter)" 또는 "얼티밋 스킬" 발동 시

- 해당 캐릭터는 **일반 행동 순서보다 항상 먼저 실행** (끼어들기)
- 다음 대기열 1순위에 위치, 원래 스킬을 사용하여 행동
- 복수 추가 턴 동시 등록 시 → 먼저 등록 순서 → 동시 등록이면 SPD 높은 쪽 우선
- 추가 턴은 구조적으로 **"턴"으로 분류되지 않음** → DoT 데미지, 턴 카운트 미변동
- 사망한 캐릭터는 이미 획득한 추가 턴이 있어도 **행동 불가**
- **무한 반복 방지**: 같은 캐릭터의 얼티밋은 한 사이클에 1회만 가능 (쿨타임), 최대 100회 제한

```
일반 대기열:  [A] [B] [C] [D]
                ↓ B가 얼티밋 → 끼어들기 등록
처리 순서:    [B얼티밋] → [A] → [B일반] → [C] → [D]
```

### 7-7. 효과 적용 타이밍

| 효과 | 적용 시점 |
|------|-----------|
| 스킬 피해 / 버프 / 디버프 | 시전자 **"턴 종료" 시점** 즉시 적용 |
| 버프/디버프 지속 턴 감산 | 버프 소유자의 **"턴 종료" 시점** |
| 지속 피해 (DoT) | 소유자의 **"턴 종료" 시점** |
| "전투 시작/종료" 발동 효과 | 연속 전투 시 **매 전투마다** 적용 |

### 7-8. 기존 시스템과의 비교

|  | 기존 (라운드로빈) | CTB (새 시스템) |
|--|-----------------|----------------|
| **턴 수** | 모든 캐릭터 동일하게 1턴씩 | SPD에 비례 (빠를수록 더 많은 턴) |
| **SPD의 역할** | 행동 순서만 결정 | 순서 + 턴 수 + SP 기여도 모두 결정 |
| **SP 축적 패턴** | 균등 | 빠른 캐릭터가 더 많이 기여 |
| **얼티밋 타이밍** | 비교적 균등 | 빠른 팀이 먼저 SP 모아 먼저 발동 |
| **속도 버프 효과** | 순서만 앞당겨짐 | 순서 앞당겨짐 + **턴 수 자체 증가** |
| **SPD 스탯 가치** | 보통 | **매우 높음** |

### 7-9. 전체 전투 흐름 다이어그램

```
[전투 시작 시점]
    │
    ▼
① 턴 대기열 생성 (SPD 기반 행동 간격 = 300/SPD)
    │
    ▼
② 턴 시작 시점
   → 데빌 행동 먼저
   → SP 획득 (+1, 양 진영 각각)
    │
    ▼
③ 스킬 결정
   ├─ 액티브 쿨타임 확인 → Active > Normal 우선
   └─ SP 임계치 달성 시 → 얼티밋 (추가 턴 "끼어들기" 등록)
    │
    ▼
④ 타겟 선택 → 스킬 실행
   ├─ 데미지: Atk × (1 - Def/(Atk+Def)) × 스킬배율 × 속성배율 × 크리배율
   ├─ 버프/디버프 적용
   └─ 트리거 체크 → 반격/얼티밋 끼어들기 등록
    │
    ▼
⑤ 턴 종료 시점
   ├─ DoT 발동
   └─ 버프 지속 턴 감산
    │
    ▼
⑥ 사망 판정 → 대기열 갱신 → 배틀 라운드 체크(10단위 재계산)
    │
    ├─ 전멸? → [전투 종료 시점] → 승리/패배
    └─ 계속 → ② 반복
```

## 8. SP 시스템

### 8-1. 개요

- 아군·적군 각각 **독립적인 SP 풀** 보유 (최대 10)
- **"턴 시작" 시점**에 양 진영 SP가 각각 **+1 증가** (누구의 턴이든 양쪽 모두 충전)
- 얼티밋 스킬 사용 시 해당 진영의 SP에서 캐릭터별 비용만큼 **즉시 차감**
- **라운드 종료(Battle Round 전환) 시 SP는 0으로 초기화**

### 8-2. 얼티밋 스킬 발동 방식

| 모드 | 설명 |
|------|------|
| **자동 모드** | SP 임계치 달성 즉시 자동 발동. 캐릭터별 SP 우선순위 순서로 처리. |
| **수동 모드** | 아군 얼티밋만 적용. 유저가 타이밍 직접 선택. 수동 대기 중 라운드/턴 종료 시 **SP 자동 환불**. 자동↔수동 전환 중 대기 중인 얼티밋도 즉시 환불. |

> 일반 공격과 액티브 스킬은 **항상 자동** (수동 선택 불가). 적팀도 항상 자동.

### 8-3. 얼티밋 SP 우선순위 처리

현재 방식은 **"1순위 캐릭터를 위해 SP를 아껴두는"** 설계:

```
현재 SP: 150
1순위 캐릭터 A: 얼티밋 비용 200 → 부족 → 발동 실패
2순위 캐릭터 B: 얼티밋 비용 80  → A 실패로 체크조차 안 됨
→ SP가 150이나 있어도 아무 얼티밋도 발동 안 됨
```

| 방식 | 동작 | SP 활용도 | 얼티밋 빈도 | 고비용 캐릭터 |
|------|------|-----------|-------------|---------------|
| **현재 (대기)** | 1순위 실패 시 전원 대기 | 낮음 | 적음 | 유리 |
| **변경 시 (건너뛰기)** | 1순위 실패 시 2순위로 | 높음 | 많음 | 불리 |

### 8-4. 캐릭터별 얼티밋 SP 비용

각 캐릭터마다 고유의 SP 비용 보유 (범위: **4~6**).

| 샘플 캐릭터 | SP 비용 |
|-------------|---------|
| 까미 (Attacker) | 6 |
| 아라미 (Supporter) | 5 |
| 다나 (Healer) | 4 |

### 8-5. SP 관련 스탯 & 로직

| 스탯/로직 | 설명 |
|-----------|------|
| SpStart | 전투 개시 시 보유 SP |
| SpRecovery | 추가 SP 회복량 |

## 11. 데미지 계산

### 기본 데미지 공식

```
Base Damage = Attack × (1 - Defense / (Attack + Defense))

Final Damage = ⌊ Base Damage × Skill Multiplier × Element Multiplier × Critical Multiplier ⌋
```

> 방어력 경감이 **비율 기반**으로 설계되어, Def가 높아질수록 경감 효율이 체감 감소한다.

### 예시 계산

| Attack | Defense | Base Damage 비율 | 설명 |
|--------|---------|-----------------|------|
| 300 | 150 | 300 × (1 - 150/450) = 200 | Def = Atk의 50% |
| 300 | 300 | 300 × (1 - 300/600) = 150 | Def = Atk과 동일 |
| 300 | 600 | 300 × (1 - 600/900) = 100 | Def = Atk의 2배 |

### 관통 처리
- `Penetration` 스탯은 방어력 직접 감소 효과 적용
- LogicDamagePenetration: 방어를 완전히 무시하는 별도 공식

### 추가 배율 적용 순서

```
Final Damage
  = ⌊ Base Damage
    × Skill Multiplier      (Normal/Active/Ultimate별 배율)
    × Element Multiplier    (속성 상성 1.0 또는 1.2)
    × Critical Multiplier   (크리 적중 시 CriDmgRatio)
    × BuffScaleBonus ⌋      (버프 스택 스케일 적용 시)
```

### 크리티컬

```
크리 발동 = Random(0~1) < (CriRatio - Target.CriResistRatio)
크리 데미지 = Base × CriDmgRatio × (1 - Target.CriDmgReduceRatio)
```

### 회피·명중

```
회피 성공 = Random(0~1) < (Target.Dodge - Caster.Hit)
→ 회피 성공 시 데미지 0
```

### 특수 데미지 공식 종류

| Logic | 설명 |
|-------|------|
| LogicDamageHpRatio | 대상 최대 HP 비례 데미지 |
| LogicDamageCri | 강제 크리티컬 데미지 |
| LogicDamageBuffScale | 아군 버프 수 비례 데미지 증폭 |
| LogicDamageDebuffScaleTarget | 대상 디버프 수 비례 데미지 증폭 |
| LogicDamageCriApplyDebuff | 크리 적중 시 디버프 부여 데미지 |

## 12. 속성 상성

### 상성 관계

```
3원소 순환:  Fire > Forest > Water > Fire  (우위 속성 공격 시 1.2배)
특수 관계:   Light ↔ Dark  (서로 공격 시 양방향 1.2배)
```

|----------------------|------|

| Water → Fire | **1.2배** |

## 13. 버프 / 디버프 시스템

### 버프 중첩 규칙

| 경우 | 처리 |
|------|------|
| **서로 다른 스킬(출처)** 의 동일 스탯 버프 | **합산 적용** (무한 중첩) |

- 버프/디버프의 남은 턴 수는 **해당 효과가 걸려있는 캐릭터의 턴 종료 시점**에 1씩 차감

### DoT (지속 피해/회복)

> **적용 시점**: 버프 소유자의 **"턴 종료" 시점**

| Logic | 설명 |
|-------|------|
| LogicDot | Atk 기반 지속 데미지 |
| LogicDotDamageHpRatio | 최대 HP% 지속 데미지 |
| LogicDotHeal | Atk 기반 지속 회복 |
| LogicDotHealHpRatio | 최대 HP% 지속 회복 |

### 동적 속도 변화 (Dynamic CTB 연동)

속도(SPD) 버프/디버프 부여·해제 **즉시** 현재 남은 목표 거리를 재계산:
```
새로운 도달 시간 = (남은 시간 × 이전 속도) / 새로운 속도
```
→ 속도 버프 = 다음 턴이 즉시 앞당겨짐
→ 속도 디버프 = 다음 턴이 즉시 뒤로 밀림

| 카라라트리 | 기절/방깎 | 화상 스택 비례 기절(Stun) |
| 라가라자 | 화상 반격 탱커 | 피격 시 화상 반격 |
| 시트리 | 속도 버퍼 | 전체 SPD 증폭 |

- 핵심 시너지: `구미호/라가라자 화상 스택 → 카라라트리 기절 → 카인 폭딜`
- 액티브만으로도 극강 딜 시너지로 **29 타임라인 승리**
- 지정 순서 얼티밋 폭격 시 → **13.0 타임라인만에 완벽 승리 (최단 기록)**

### 17-3. 밸런스 주의사항

#### ① SPD 스탯 가치 급상승
CTB 시스템에서 SPD는 단순 순서 결정이 아니라:
1. **행동 횟수 자체를 결정** (SPD 100 → 3턴, SPD 50 → 1턴)
2. **더 많은 행동 = 더 많은 SP 충전** → 얼티밋 선점
3. **더 많은 행동 = 더 많은 딜/힐**

> SPD 밸런스가 전투 지배력에 직결 → **SPD 수치 범위 엄격 관리 필요**

#### ② 속도 버프/디버프의 영향 극대화

| 효과 | 결과 |
|------|------|
| 속도 2배 버프 | 남은 턴 수 2배 + SP 충전 2배 + 얼티밋 빈도 2배 |
| 속도 절반 디버프 | 남은 턴 수 절반 → 사실상 행동 박탈 |

> 속도 관련 스킬의 영향력이 기존 시스템보다 훨씬 크므로 **밸런스 점검 필수**

#### ③ 몬스터 광역기 난이도 급상승

몬스터에게 광역기 부여 시 파티 난이도가 급격히 상승.
유저는 **공용 SP를 전략적 타이밍에 사용해 핵심 버프/디버프(도발·화상·속도업)를 순서대로 발동**시키는 것이 이 전투 시스템의 핵심 재미 요소.

#### ④ 요약 점검 항목

| 항목 | 내용 | 중요도 |
|------|------|--------|
| SPD 밸런스 | 속도 높을수록 턴 수 + 얼티밋 선점 + SP 기여 모두 유리 | **높음** |
| 얼티밋 SP 우선순위 | 고비용 캐릭터 앞에 있으면 저비용도 발동 못 하는 구조 | 중간 |
| 속도 버프/디버프 | 턴 수를 급격히 바꿔 기존보다 영향력 큼 | 중간 |
| 빠른 캐릭터 연속 행동 | 일반 → 얼티밋 → 일반 패턴은 의도된 동작 | 낮음 (인지만) |
